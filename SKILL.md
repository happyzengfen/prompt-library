# Prompt Library - 提示词库分发器

一个单一 skill 的提示词分发仓库：从 `references/prompts.md` 中读取 13 个预置模板，根据用户的编号、标题或意图进行匹配，并原样输出对应提示词正文。

本仓库的运行时契约保持最小且稳定：`SKILL.md` 负责定义触发、匹配与输出规则，`references/prompts.md` 是 13 个模板正文的唯一真实来源。

**触发词**：「给我提示词」「查提示词」「找提示词」「帮我拆解文章」「RSS聚合」「工作流设计」「文章总结」「项目资料提炼」「视频内容概括」「视频SOP」「爆款拆解」「字幕提取」「视听分析」「代码Spec」「研报分析」「爆款标题」「给我第X个提示词」。

**不触发**：用户已提供素材并要求直接执行任务时（如发来一篇文章说"帮我总结"），不输出提示词模板，而是直接执行任务。也不用于优化提示词、创建新提示词等场景。

## Goal

用户说出需求关键词、标题线索或编号后，立即输出 `references/prompts.md` 中对应的提示词模板原文，供用户直接复制使用。

## When to use

- 用户要求获取、查看、查找某个提示词模板
- 用户用自然语言描述场景（如"我要拆解一篇爆款文章"），匹配到对应模板
- 用户指定编号（如"给我第5个"、"项目资料那个"）
- 用户说"有哪些提示词"、"列出所有提示词"

## Do not use

- 用户已提供具体素材（文章、视频文案、代码需求等）并要求执行任务——此时直接做任务，不输出模板
- 用户要创建新提示词或优化现有提示词——应改用专门的提示词设计/优化能力，而不是本分发 skill
- 用户在做编程、调试、文件操作等与提示词无关的工作

## Procedure

1. 读取 `references/prompts.md` 获取全部 13 个提示词模板；该文件是模板正文的唯一真实来源
2. 根据用户输入进行意图匹配：
   - 如果用户指定了编号（如"第3个"），直接定位
   - 如果用户用自然语言描述（如"视频字幕"），按关键词匹配对应模板
   - 如果用户说"列出所有"或"有哪些"，输出13个模板的编号+标题清单
3. 输出匹配结果

## Output format

先输出匹配结果标题，再用 Markdown 代码块包裹模板正文。示意如下：

    ### 匹配结果：#{编号} {标题}

    ```
    {提示词模板原文，完整输出，不做任何修改}
    ```

- 用 Markdown 代码块包裹提示词内容，方便用户一键复制
- 先说明匹配到的是哪个（编号+标题），再输出内容
- 如果匹配不确定，列出最接近的2-3个选项让用户选择

## Definition of done

- [ ] 输出的提示词内容与 references/prompts.md 中完全一致，无任何篡改
- [ ] 正确匹配到用户意图对应的提示词编号
- [ ] 匹配失败时列出全部13个标题供用户选择
- [ ] 输出格式为 Markdown 代码块，可直接复制

## Failure handling

- 无法匹配：输出13个提示词的编号+标题清单，让用户选择
- 多个可能匹配：列出候选项（最多3个），让用户确认
- 用户已提供素材要求执行：不输出模板，直接使用对应模板的逻辑执行任务
