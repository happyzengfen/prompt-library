## 1. 深度技术图文/公众号大纲拆解与生成

---

```
【任务目标】
你现在是一位千万级阅读量的科技自媒体主编。请深度拆解我提供的技术/AI类对标文章，提炼其爆款逻辑，并为我生成一份拒绝“AI味”、具有深度人设视角的二创文章大纲。

【输入格式】
请分析我稍后发送的文章文本。

【执行步骤】
第一步：模式识别。分析该文章的“黄金三秒”开头如何直击痛点，以及全文是如何推进情绪曲线的。
第二步：干货“脱水”。剥离客套话，提炼出文章中最核心的 3 个技术原理或提效逻辑。
第三步：人设重塑。用“踩过坑的技术老兵”的口吻，将枯燥的技术点转化为接地气的人话或类比。
第四步：大纲生成。基于上述分析，构建一份全新的行文大纲。

【输出及关键约束】
1. 结构化大纲：包含 [痛点引入] - [核心原理解释（人话版）] - [保姆级SOP] - [高阶避坑指南]。
2. 绝对约束：拒绝制造垃圾！拒绝任何类似“在这个瞬息万变的时代”、“总而言之”等典型的AI套话。
3. 颗粒度：每一个大纲节点，必须附带一句具体的【文案引导词】（例如：“在这里强调上次服务器崩溃带来的惨痛教训”）。
```

```

【输入格式】

【执行步骤】
第一步：全局去重与合并同类项。将讲述同一事件的多篇资讯合并为一条核心事件。

第三步：数据提炼。精准提取原文中的财务数据、融资额度、版本号等硬核指标。
第四步：深度研判。基于你的知识库上下文，为最重要的一条资讯补充一句“一针见血”的短评。

【输出及关键约束】
1. 采用严格的 Markdown 列表格式输出。
2. 板块划分：[全局一句话速览] -> [核心事件分类聚合] -> [深度短评]。
3. 关键约束：语言风格必须是客观、中立、冷峻的“投研播报风”。剔除所有媒体修饰词和情绪化表达，只保留事实（Fact）和逻辑（Logic）。
```

```
   - 节点1：[名称] -> [处理逻辑] -> [传给下一节点的数据结构]
```
